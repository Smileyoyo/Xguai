# 博客部署指南

恭喜！你的博客已经搭建完成。下面是部署到 GitHub Pages 的步骤：

## 📋 部署前准备

### 1. 修改配置文件

在部署之前，需要修改以下配置：

#### 编辑 `_config.yml` 文件：

将以下内容中的 `username` 替换为你的 GitHub 用户名：

```yaml
# URL 部分
url: https://username.github.io

# 部署部分
deploy:
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: main
```

#### 编辑 `_config.butterfly.yml` 文件：

```yaml
# 侧边栏作者卡片
card_author:
  button:
    link: https://github.com/username
```

### 2. 创建 GitHub 仓库

1. 登录 [GitHub](https://github.com/)
2. 创建新仓库，仓库名必须为：`username.github.io`（username 替换为你的用户名）
3. 仓库权限设置为 **Public**（公开）
4. 不要初始化 README、.gitignore 等

### 3. 配置 Git（如果还没有配置）

```bash
# 配置用户名
git config --global user.name "Your Name"

# 配置邮箱
git config --global user.email "your.email@example.com"
```

## 🚀 开始部署

### 方法一：使用 Hexo 部署命令

1. **安装依赖**（如果还没有安装）

```bash
npm install
```

2. **生成并部署**

```bash
npx hexo clean
npx hexo generate
npx hexo deploy
```

或者一次性执行：

```bash
npx hexo clean && npx hexo generate && npx hexo deploy
```

3. **首次部署会提示输入 GitHub 凭据**

当执行 `hexo deploy` 时，会要求输入 GitHub 用户名和密码（或 Personal Access Token）。

> **重要提示**：GitHub 已经不支持密码验证，需要使用 Personal Access Token。

### 获取 GitHub Personal Access Token

1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token** → **Generate new token (classic)**
3. 设置 token 名称（如：hexo-deploy）
4. 选择权限：
   - `repo`（完整仓库访问权限）
   - `workflow`（如果需要）
5. 点击 **Generate token**
6. **复制生成的 token**（只显示一次，请妥善保存）

然后在部署时：
- Username: 你的 GitHub 用户名
- Password: 粘贴刚才的 Token

### 方法二：手动部署（可选）

如果你想手动管理仓库：

1. **初始化 Git 仓库**

```bash
git init
git add .
git commit -m "Initial commit"
```

2. **关联远程仓库**

```bash
git remote add origin https://github.com/username/username.github.io.git
git branch -M main
git push -u origin main
```

注意：使用此方法需要单独创建 `username.github.io` 仓库来存放生成的 `public/` 目录内容。

## ✅ 访问你的博客

部署完成后，等待 1-2 分钟，访问：

```
https://username.github.io
```

将 `username` 替换为你的 GitHub 用户名。

## 📝 日常使用

### 创建新文章

```bash
# 方式一：使用 npm script
npm run new "文章标题"

# 方式二：直接使用 hexo 命令
npx hexo new "文章标题"
```

文章会创建在 `source/_posts/` 目录下。

### 本地预览

```bash
# 启动本地服务器
npm run server
# 或
npx hexo server

# 访问 http://localhost:4000
```

### 发布新内容

```bash
# 清除缓存、生成静态文件、部署
npx hexo clean && npx hexo generate && npx hexo deploy
```

## 🎨 自定义博客

### 修改网站信息

编辑 `_config.yml`：

```yaml
# Site
title: 你的博客标题
subtitle: 副标题
description: 描述
author: 你的名字
```

### 主题配置

编辑 `_config.butterfly.yml` 来自定义主题。

详细配置请参考：[Butterfly 主题文档](https://butterfly.js.org/)

### 添加图片

1. 将图片放在 `source/img/` 目录
2. 在文章中使用：

```markdown
![图片描述](/img/图片名.png)
```

## 🔧 常见问题

### 问题 1：部署后 404

- 检查仓库名是否正确（必须是 `username.github.io`）
- 等待几分钟，GitHub Pages 需要时间生效
- 检查仓库是否设置为 Public

### 问题 2：样式错乱

运行清除缓存命令：

```bash
npx hexo clean
npx hexo generate
```

### 问题 3：图片不显示

- 确保图片放在 `source/img/` 目录
- 在文章中使用 `/img/图片名.png` 格式引用

### 问题 4：部署失败

- 检查网络连接
- 确认 GitHub Token 是否有效
- 检查仓库地址是否正确
