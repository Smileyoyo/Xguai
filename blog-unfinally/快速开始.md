# 🚀 方案 B：7 个独立飞书机器人 - 快速开始

## 📋 步骤概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 1.创建应用  │ →  │ 2.配置权限  │ →  │ 3.填入凭证  │ →  │ 4.启动服务  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

---

## 第一步：创建 7 个飞书应用（15 分钟）

访问 https://open.feishu.cn/app

### 逐个创建以下应用：

| 序号 | 应用名称 | 内部应用名 | 端口号 |
|:----:|---------|-----------|--------|
| 1 | 🤖 CEO 助手 | `virtual-company-ceo` | 18790 |
| 2 | 📋 产品助手 | `virtual-company-product` | 18791 |
| 3 | 💻 开发助手 | `virtual-company-dev` | 18792 |
| 4 | 🎨 设计助手 | `virtual-company-design` | 18793 |
| 5 | 📢 市场助手 | `virtual-company-marketing` | 18794 |
| 6 | 📊 数据助手 | `virtual-company-data` | 18795 |
| 7 | 🔧 运维助手 | `virtual-company-ops` | 18796 |

### 每个应用的配置步骤：

1. **创建应用**
   - 点击「创建企业自建应用」
   - 填写名称和描述
   - 上传对应 emoji 作为头像

2. **添加能力**
   - 「添加应用能力」→ 开启「机器人」

3. **申请权限**
   - `im:message:send_as_bot`
   - `im:message.group_msg`
   - `im:chat:readonly`

4. **事件订阅**
   - 配置方式：长连接
   - 订阅事件：`im.message.receive_v1`

5. **发布应用**
   - 「版本管理与发布」→ 创建版本 → 申请发布
   - 等待管理员审批

## 第三步：运行配置脚本（2 分钟）

在终端中执行：

```bash
cd workspace/feishu-apps
./configure.sh
```

按提示依次输入 7 个应用的 App ID 和 App Secret。

脚本会自动：
- 替换所有配置文件中的占位符
- 生成 `start-all.sh` 启动脚本
- 生成 `stop-all.sh` 停止脚本

---

## 第四步：创建飞书群并拉入机器人（5 分钟）

1. 在飞书创建群聊
   - 群名称：`虚拟公司 · AI 协作空间`

2. 将 7 个机器人逐个拉入群聊
   - 点击群设置 → 添加群成员
   - 搜索机器人名称（如「CEO 助手」）
   - 点击添加

---

## 第五步：启动所有机器人（1 分钟）

```bash
./start-all.sh
```

启动后，可以通过以下地址检查状态：
- CEO: http://localhost:18790/health
- 产品部: http://localhost:18791/health
- 开发部: http://localhost:18792/health
- 设计部: http://localhost:18793/health
- 市场部: http://localhost:18794/health
- 数据部: http://localhost:18795/health
- 运维部: http://localhost:18796/health

## 🆘 故障排查

### 机器人没有响应

1. 检查服务是否启动
   ```bash
   curl http://localhost:18790/health
   ```

2. 检查是否在群里 @了机器人（需要带完整名称）

3. 检查应用是否已审批发布

4. 检查事件订阅是否配置正确

### 端口冲突

如果端口被占用，可以修改对应目录下 `openclaw.json` 中的 `gateway.port` 值。
